<objective>
Bring the three most structurally incomplete domain pages (chargeback, retrieval, adjustment) to visual parity with the authorization reference implementation.

Purpose: These three pages use Ant Design `Statistic` inside raw `Card` for KPI display, which produces visually inconsistent output compared to the shared `KPICard` component used by authorization/settlement/funding. Converting them to `KPICard` satisfies UX-01 (consistent layout). Replacing `<Spin>` with `<Skeleton>` satisfies UX-03. Note: the Adjustment page has no chart API endpoint (only kpis + details) — it stays KPIs-only but with the correct component. Do not add chart content that calls non-existent endpoints.

Output: Three updated domain page files. All KPI metrics rendered via KPICard with loading prop. Chart Spin overlays replaced with Card+Skeleton. Formatter imports from @/lib/formatters.
</objective>
