    **Common bug patterns and fixes:**

    PATTERN A — Column name mismatch (most common cause of zeros in KPI cards):
    - Symptom: curl returns success: true but data values are all 0 or null
    - Diagnosis: Read the failing route.ts file. Look at the SELECT column aliases and compare them to what the TypeScript type expects.
    - Fix: In the route.ts file, update the SQL column aliases to match what the frontend type expects. Example: if route selects `COUNT(*) as count` but frontend type reads `totalTransactions`, change to `COUNT(*) as "totalTransactions"`.

    PATTERN B — Date column name wrong (causes empty results):
    - Symptom: endpoint returns success: true with empty data array
    - Diagnosis: Check the WHERE clause date filter. The column name may be wrong for that MARTS table.
    - Fix: Update the date column name in the WHERE clause. Authorization/kpis uses `transaction_date`; Settlement uses `settlement_date`; Deposits uses `deposit_date`; Chargebacks uses `chargeback_date`; Retrievals uses `retrieval_date`; Adjustments uses `adjustment_date`.

    PATTERN C — CLNT_ID filter blocks all rows:
    - Symptom: Query returns zero rows even though Snowflake Web UI shows data for the date range
    - Diagnosis: The route appends `AND CLNT_ID = 'dmcl'` (or similar RLS filter). The actual CLNT_ID value in MARTS may differ.
    - Fix: Check the actual CLNT_ID values in one MARTS table: `SELECT DISTINCT CLNT_ID FROM COCO_SDLC_HOL.MARTS.AUTHORIZATIONS LIMIT 5`. Update the hardcoded CLNT_ID value in the failing routes to match.

    PATTERN D — SQL syntax error causing 503:
    - Symptom: endpoint returns SNOWFLAKE_CONNECTION_ERROR with a SQL syntax error in details
    - Diagnosis: Read the full error in the curl response details field
    - Fix: Correct the SQL in the route.ts file

    For each bug fixed:
