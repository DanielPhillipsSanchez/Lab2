  <name>Task 2: Push image to Snowflake registry and run setup.sql</name>
  <files></files>
  <action>
Human must complete Docker push and Snowflake provisioning — these require authentication to external systems that Claude cannot access.

**Step 1 — Authenticate Docker with Snowflake registry:**
```bash
snow spcs image-registry login --connection ennovate
```

**Step 2 — Get repository URL:**
In a Snowflake worksheet, run:
```sql
USE ROLE SYSADMIN;
SHOW IMAGE REPOSITORIES IN SCHEMA COCO_SDLC_HOL.PUBLIC;
```
Note the `repository_url` column value (format: `orgname-acctname.registry.snowflakecomputing.com/coco_sdlc_hol/public/coco_sdlc_hol_repo`).

**Step 3 — Tag and push:**
```bash
docker tag coco-portal:latest <REPO_URL>/coco-portal:latest
docker push <REPO_URL>/coco-portal:latest
```

**Step 4 — Fill in setup.sql placeholders:**
Open `setup.sql` and replace:
- `<PASTE YOUR UNENCRYPTED PEM KEY CONTENT HERE>` — paste the full PEM content from `c:/Users/TrentFoley/Keys/ennovate-trent-foley.p8`
- `<SNOWFLAKE_ACCOUNT>` — your account identifier
- `<SNOWFLAKE_USER>` — your Snowflake username
- `<REPO_URL>` — repository_url from step 2

**Step 5 — Run setup.sql in a Snowflake worksheet.**
After `CREATE COMPUTE POOL`, run `SHOW COMPUTE POOLS` and wait for `state = ACTIVE` or `IDLE` before running `CREATE SERVICE` (2–5 minutes). If `STANDARD_1` fails, use `CPU_X64_XS` as the instance family.

**Step 6 — Wait for RUNNING state:**
```sql
SHOW SERVICES LIKE 'COCO_SDLC_HOL_SERVICE';
```

**Step 7 — Get the public endpoint URL:**
```sql
SHOW ENDPOINTS IN SERVICE coco_sdlc_hol_service;
```
Note the `ingress_url` value.
