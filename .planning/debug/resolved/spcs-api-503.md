## Resolution

root_cause: SPCS enforces OAuth-only authentication for all outbound Snowflake SDK connections from inside containers (error code 395090). The app was using JWT key-pair auth (SNOWFLAKE_JWT authenticator), which SPCS blocks regardless of key validity. SPCS injects a rotating OAuth token at /snowflake/session/token that must be used instead.

fix: |
  Modified apps/frontend/src/lib/snowflake.ts:
  - Added SPCS_TOKEN_PATH = '/snowflake/session/token' constant
  - In createConnection(): Added SPCS OAuth as priority auth strategy (reads token file, sets authenticator: 'OAUTH', sets token: <file content>)
  - In getConfig(): Auth validation now skips the "missing auth" error when SPCS token file exists
  - In isConfigured(): Returns true when SPCS token file exists (even without SNOWFLAKE_PRIVATE_KEY)

  Modified apps/frontend/src/app/api/cortex/chat/route.ts:
  - Added SPCS_TOKEN_PATH constant
  - Added getAuthToken() function that returns SPCS OAuth token when token file exists, JWT otherwise
  - Updated isSnowflakeConfigured() to return true when SPCS token file present
  - Updated fetch() call to use getAuthToken() result with dynamic X-Snowflake-Authorization-Token-Type header
